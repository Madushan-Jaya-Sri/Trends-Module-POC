### Component Details:

#### Summary Statistics Cards
- **Total Trends:** Total number of trends analyzed
- **Google Trends:** Number of Google Trends items
- **YouTube Trends:** Number of YouTube videos
- **TikTok Trends:** Number of TikTok items
- Each card shows count and percentage

#### Score Methodology Info
- Collapsible section (collapsed by default)
- Shows weight distribution and explanation
- Helps users understand the scoring system

#### Sorting Options
- Dropdown with options:
  - Trending Score (High to Low) [Default]
  - Trending Score (Low to High)
  - Volume Score
  - Engagement Score
  - Velocity Score
  - Recency Score
  - Most Recent
  - Platform (Google â†’ YouTube â†’ TikTok)

#### Trending Item Card
- **Rank Number:** #1, #2, etc.
- **Platform Badge:** Color-coded badge (Google/YouTube/TikTok)
- **Title/Name:** Trend title or query
- **Overall Score:** Large, prominent number (0-100)
- **Score Breakdown:** Visual bars or numbers for each component
- **Metadata:** Platform-specific stats
- **Timestamp:** When it started trending
- **Actions:**
  - View Details (expands card)
  - Go to Platform (opens external link)

## 7. RESPONSIVE DESIGN NOTES

### Desktop (> 1024px)
- Grid view: 3-4 cards per row
- Full sidebar filters visible
- All data fields shown

### Tablet (768px - 1024px)
- Grid view: 2 cards per row
- Collapsible sidebar filters
- Some metadata hidden in collapsed state

### Mobile (< 768px)
- List view only (1 card per row)
- Bottom drawer for filters
- Minimal metadata shown
- Click to expand for full details

## 9. KEY INTERACTIONS

### Filter Application
1. User selects filters
2. "Apply" or auto-apply on change
3. Show loading state
4. Fetch new data
5. Update all sections
6. Scroll to top

### Card Expansion
1. User clicks "View Details"
2. Card expands with animation
3. Show full score breakdown
4. Show detailed metadata
5. Provide action buttons

### Platform Navigation
1. User clicks platform link
2. Open in new tab
3. Keep current state

### Sort & View Changes
1. Instant client-side sorting
2. Smooth transition animations
3. Maintain scroll position if possible
